# Pear — Vision & Positioning

> Source: pearcode.dev (March 2026)

---

## One-Liner

**The learning engine for AI-accelerated builders.**

---

## The Problem

**AI is making engineers faster. Not better.**

1. **You're shipping code you don't understand.** AI tools optimize for output, not comprehension.
2. **Nobody has time to teach you anymore.** Senior engineers are overwhelmed. Mentorship is broken.
3. **Learning happens somewhere else. It shouldn't.** Context-switching kills learning momentum.

### The Data

- 19% slower with AI (METR, 2025)
- 67% drop in junior roles (Stanford)
- 66% spend time fixing flawed AI code (Stack Overflow)
- 4x more code duplication (GitClear)

---

## The Core Promise

**You ship code. Pear teaches you what it means.**

Any LLM can give you an answer. Pear makes sure you never need to ask the same question twice.

## Five Moat Pillars

| Pillar | What It Does |
|---|---|
| **Learning state memory** | Pear remembers what you know |
| **Concept graph** | Pear maps how concepts connect |
| **Adaptive pedagogy** | Pear changes how it teaches you |
| **Intervention timing** | Pear teaches at the right moment |
| **Skill progression** | Pear shows you how far you've come |

## Target Audiences

- Self-taught developers
- Bootcamp grads
- Junior developers using AI tools
- Founders building their own product
- PMs/designers who code
- Senior engineers switching stacks

**Primary audience:** Devs who ship with AI but want to understand their code. Self-selecting learners — not a job title.

## Key Differentiators

- **A learning system, not an explainer** — must scream on site
- **Socratic > informational** — questions beat explanations
- **Teach in the gaps** — don't break flow, intervene during natural pauses
- **Own the learning positioning** — users self-select for learning
